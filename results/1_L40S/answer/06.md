# Prompt
Node.js 앱을 위한 Dockerfile을 작성해줘.

# Answer
물론입니다! 아래는 Node.js 애플리케이션을 위한 기본적인 `Dockerfile` 예시입니다. 이 파일은 일반적인 Node.js 앱에 사용할 수 있으며, 필요에 따라 수정하거나 확장할 수 있습니다.

### 📄 Dockerfile 예시

```Dockerfile
# 사용할 베이스 이미지 (Node.js 공식 이미지 사용)
FROM node:18-alpine

# 작업 디렉토리 설정
WORKDIR /app

# 의존성 파일 복사 (package.json과 package-lock.json)
COPY package*.json ./

# 의존성 설치
RUN npm ci --only=production

# 애플리케이션 소스 코드 복사
COPY . .

# 포트 설정 (기본적으로 3000 포트 사용)
EXPOSE 3000

# 애플리케이션 실행 명령어
CMD ["node", "server.js"]
```

---

### ✅ 설명:

- `FROM node:18-alpine`: 가볍고 빠른 Alpine 기반의 Node.js 18 이미지를 사용합니다.
- `WORKDIR /app`: 컨테이너 내에서 작업할 디렉토리를 설정합니다.
- `COPY package*.json ./`: 패키지 파일을 복사하여 의존성을 설치합니다.
- `RUN npm ci --only=production`: 프로덕션 환경에 필요한 의존성만 설치합니다.
- `COPY . .`: 애플리케이션 코드 전체를 복사합니다.
- `EXPOSE 3000`: 컨테이너가 사용할 포트를 표시합니다.
- `CMD ["node", "server.js"]`: 애플리케이션을 실행하는 명령어입니다. `server.js`는 실제 애플리케이션의 진입점입니다.

---

### 🧪 추가 팁:

- 만약 `npm install` 대신 `npm ci`를 사용하고 싶다면, `package-lock.json`이 반드시 있어야 합니다.
- `alpine` 이미지는 가볍지만, 일부 패키지가 호환되지 않을 수 있으므로 필요 시 `node:18` (Debian 기반)으로 변경 가능합니다.

---

필요하다면 `docker-compose.yml`이나 `.dockerignore` 파일도 도와드릴 수 있어요!
