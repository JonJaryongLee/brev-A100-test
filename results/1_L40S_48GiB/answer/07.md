# Prompt
프로세스와 스레드의 차이점을 코드 레벨에서 설명해줘.

# Answer
프로세스(Process)와 스레드(Thread)는 운영체제에서 프로그램을 실행하는 두 가지 기본 단위입니다. 이 둘은 **실행 단위**로서의 역할은 비슷하지만, **메모리 공간과 자원 공유 방식**에서 큰 차이가 있습니다.

---

## 🔹 1. 정의

### ✅ 프로세스 (Process)
- 실행 중인 프로그램의 인스턴스.
- 각각 **독립된 메모리 공간**을 가진다.
- CPU 시간, 파일 디스크립터, 환경 변수 등 **자원을 독립적으로 관리**한다.

### ✅ 스레드 (Thread)
- 프로세스 내부에서 실행되는 **최소 실행 단위**.
- 같은 프로세스 내의 다른 스레드들과 **메모리 공간과 자원을 공유**한다.

---

## 🔹 2. 코드 레벨에서의 차이 예시

### 🧪 예시: Python에서 프로세스와 스레드 비교

#### 1. **스레드 사용 (threading 모듈)**

```python
import threading
import time

def worker(name, delay):
    for i in range(5):
        print(f"{name}: {i}")
        time.sleep(delay)

# 스레드 생성
t1 = threading.Thread(target=worker, args=("Thread-1", 1))
t2 = threading.Thread(target=worker, args=("Thread-2", 1))

# 스레드 시작
t1.start()
t2.start()

# 스레드 종료 대기
t1.join()
t2.join()
```

- `t1`과 `t2`는 같은 프로세스 내에서 실행되며, **공유 메모리 공간**을 사용합니다.
- `print()` 같은 출력은 **동일한 메모리 공간**에서 처리되므로, **출력 순서가 뒤섞일 수 있습니다** (race condition).

#### 2. **프로세스 사용 (multiprocessing 모듈)**

```python
import multiprocessing
import time

def worker(name, delay):
    for i in range(5):
        print(f"{name}: {i}")
        time.sleep(delay)

# 프로세스 생성
p1 = multiprocessing.Process(target=worker, args=("Process-1", 1))
p2 = multiprocessing.Process(target=worker, args=("Process-2", 1))

# 프로세스 시작
p1.start()
p2.start()

# 프로세스 종료 대기
p1.join()
p2.join()
```

- `p1`과 `p2`는 **각각 독립된 메모리 공간**에서 실행됩니다.
- 따라서 **메모리 공유가 없고**, 서로 영향을 주지 않습니다.

---

## 🔹 3. 주요 차이 요약

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 | 공유 |
| 자원 공유 | 없음 | 공유 |
| 생성/종료 비용 | 높음 | 낮음 |
| 컨텍스트 스위칭 | 비쌈 | 싸움 |
| 병렬성 | 실제 병렬 처리 가능 (다중 코어) | 동일 프로세스 내에서 동시성 |
| 예시 언어 | C, Python (multiprocessing) | Python (threading) |

---

## 🔹 4. 실생활 예시

- **웹 서버**: 여러 요청을 동시에 처리하기 위해 **프로세스 또는 스레드**를 사용.
  - 예: Apache는 각 요청마다 **프로세스**를 생성.
  - 예: Node.js는 **단일 스레드**로 여러 요청을 처리 (비동기).
- **게임 엔진**: 게임 로직, 오디오, 그래픽 등을 **스레드**로 분리하여 병렬 처리.

---

## ✅ 결론

- **스레드**는 **같은 프로세스 내에서 자원을 공유하며**, **작은 단위의 병렬 처리**에 적합.
- **프로세스**는 **완전히 독립된 메모리 공간**을 가지며, **안정적인 병렬 처리**에 유리.

코드 레벨에서는 `threading`과 `multiprocessing` 모듈을 통해 직접적으로 구현해볼 수 있으며, 성능과 자원 사용량 측면에서 선택이 달라질 수 있습니다.
