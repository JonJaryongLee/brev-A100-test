# Prompt
PostgreSQL 데이터베이스 성능 최적화 방법을 알려줘.

# Answer
PostgreSQL 데이터베이스의 성능을 최적화하는 것은 애플리케이션의 응답 속도, 처리량, 자원 사용 효율성을 높이는 데 매우 중요합니다. 아래는 PostgreSQL 성능 최적화를 위한 주요 방법들을 정리한 것입니다:

---

## 🔧 1. **인덱스(Index) 최적화**

### ✅ 인덱스 생성 전략
- 자주 사용되는 `WHERE`, `JOIN`, `ORDER BY` 절에 있는 컬럼에 인덱스를 생성하세요.
- 예: `SELECT * FROM users WHERE email = 'user@example.com';`

```sql
CREATE INDEX idx_users_email ON users(email);
```

### ✅ 복합 인덱스 (Composite Index)
- 여러 컬럼을 함께 사용하는 쿼리에 유용합니다.

```sql
CREATE INDEX idx_users_email_status ON users(email, status);
```

### ⚠️ 주의사항
- 인덱스는 **읽기 성능 향상**은 하지만, **쓰기(INSERT/UPDATE/DELETE)** 성능은 저하될 수 있습니다.
- 불필요한 인덱스는 제거하세요.

---

## 📊 2. **쿼리 최적화**

### ✅ EXPLAIN ANALYZE
쿼리 실행 계획을 확인하여 성능 문제를 파악하세요.

```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'user@example.com';
```

### ✅ WHERE 조건 최적화
- 인덱스가 있는 컬럼을 기준으로 필터링하세요.
- `LIKE '%value'` 같은 패턴은 인덱스를 사용하지 못합니다.

### ✅ JOIN 최적화
- JOIN 시에는 관련 컬럼에 인덱스가 있어야 합니다.
- `INNER JOIN`보다 `EXISTS`를 사용하면 성능이 개선될 수 있습니다.

---

## 🛠️ 3. **설정 최적화 (postgresql.conf)**

### ✅ 주요 설정 항목

| 설정 | 설명 |
|------|------|
| `shared_buffers` | PostgreSQL이 사용할 메모리 양 (권장: 전체 RAM의 25%) |
| `effective_cache_size` | OS 캐시 포함한 총 메모리 (권장: 전체 RAM의 50~75%) |
| `work_mem` | 각 작업에서 사용할 메모리 (쿼리당) |
| `maintenance_work_mem` | VACUUM, ALTER TABLE 등 작업 시 사용할 메모리 |
| `checkpoint_completion_target` | 체크포인트 시간 조절 (0.8~0.9 권장) |

예:
```conf
shared_buffers = 2GB
work_mem = 64MB
effective_cache_size = 6GB
```

> ⚠️ 변경 후 PostgreSQL 재시작 필요

---

## 🧹 4. **VACUUM 및 ANALYZE**

### ✅ VACUUM
- 삭제된 행을 정리하고, 공간을 회수합니다.

```sql
VACUUM users;
```

### ✅ ANALYZE
- 테이블 통계 정보를 업데이트하여 쿼리 계획을 최적화합니다.

```sql
ANALYZE users;
```

### ✅ 자동 VACUUM 설정
```conf
autovacuum = on
autovacuum_max_workers = 3
autovacuum_naptime = 1min
```

---

## 🗃️ 5. **테이블 분할 (Partitioning)**

대용량 테이블의 경우, **파티셔닝**을 통해 성능을 향상시킬 수 있습니다.

### 예: 월별 파티셔닝

```sql
CREATE TABLE logs_2025_04 (
    CHECK ( created_at >= '2025-04-01' AND created_at < '2025-05-01' )
) INHERITS (logs);

CREATE INDEX idx_logs_2025_04_created_at ON logs_2025_04(created_at);
```

---

## 🔄 6. **캐싱 전략**

- **Redis 또는 Memcached**와 같은 외부 캐시 시스템을 활용해 자주 조회되는 데이터를 캐싱하세요.
- DB 쿼리를 줄이는 것이 가장 효과적인 방법 중 하나입니다.

---

## 📈 7. **모니터링 도구 활용**

- **pg_stat_statements**: 자주 실행되는 쿼리 추적
- **pgBadger**, **pgHero**, **New Relic** 등으로 성능 분석 가능

```sql
-- pg_stat_statements 활성화
LOAD 'pg_stat_statements';
```

---

## 🧪 8. **테스트 및 벤치마킹**

- 실제 환경에서 **부하 테스트**(예: `pgbench`, `sysbench`)를 통해 성능을 측정하세요.
- 쿼리 변경 후 성능 비교를 통해 최적화 여부를 확인하세요.

---

## 🧠 요약

| 항목 | 최적화 방법 |
|------|-------------|
| 인덱스 | 적절한 컬럼에 인덱스 생성 |
| 쿼리 | EXPLAIN로 최적화 |
| 설정 | `shared_buffers`, `work_mem` 등 조정 |
| 정리 | 주기적으로 `VACUUM`, `ANALYZE` |
| 파티셔닝 | 대용량 테이블에 적용 |
| 캐싱 | 외부 캐시 시스템 활용 |
| 모니터링 | `pg_stat_statements` 등을 활용 |

---

필요하다면 특정 쿼리나 상황에 맞춘 최적화 조언도 드릴 수 있습니다. 😊
