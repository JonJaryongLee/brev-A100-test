# Prompt
Rust에서 메모리 관리는 어떻게 이루어지는지 설명해줘.

# Answer
Rust는 **메모리 관리**를 컴파일 타임에 처리하여, 런타임 오버헤드 없이 안전한 메모리 사용을 보장합니다. 이는 Rust의 핵심 철학 중 하나인 **"안전성과 효율성"**을 반영합니다.

---

## 🔑 Rust의 메모리 관리 원칙

Rust는 다음과 같은 방식으로 메모리 관리를 합니다:

1. **소유권 시스템 (Ownership System)**
2. **대여 시스템 (Borrowing System)**
3. **생명주기 (Lifetime)**
4. **RAII (Resource Acquisition Is Initialization)**

---

## 1. 소유권 시스템 (Ownership)

### 핵심 규칙:
- 각 값은 **하나의 소유자**만 가질 수 있다.
- 소유자는 **하나의 값만 소유**할 수 있다.
- 값이 범위를 벗어나면 자동으로 해제된다 (drop).

### 예시:
```rust
let s1 = String::from("Hello");
let s2 = s1; // s1은 s2로 이동됨 (move)
// println!("{}", s1); // 컴파일 에러! s1은 더 이상 유효하지 않음
```

> `s1`이 `s2`로 이동되었기 때문에, `s1`은 더 이상 유효하지 않다. 이는 **소유권 이전**이다.

---

## 2. 대여 시스템 (Borrowing)

- 소유권을 넘기지 않고 **참조**를 통해 값을 사용할 수 있음.
- 참조는 **불변(immutable)** 또는 **가변(mutable)** 가능.

### 예시:
```rust
fn main() {
    let mut s = String::from("Hello");
    change(&mut s); // mutable borrow
    println!("{}", s);
}

fn change(s: &mut String) {
    s.push_str(", world!");
}
```

---

## 3. 생명주기 (Lifetime)

- Rust는 **참조가 유효한 기간**을 추적함.
- 컴파일 타임에 lifetime을 검사하여 **dangling pointer**를 방지함.

### 예시:
```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

> `'a`는 두 입력값의 참조가 같은 생명주기를 가지도록 보장함.

---

## 4. RAII (Resource Acquisition Is Initialization)

- 리소스(예: 메모리)는 객체가 생성될 때 할당되고, 소멸될 때 해제됨.
- Rust에서는 `Drop` 트레이트를 통해 자동으로 해제됨.

### 예시:
```rust
struct MyStruct {
    data: Vec<i32>,
}

impl Drop for MyStruct {
    fn drop(&mut self) {
        println!("MyStruct is being dropped");
    }
}
```

---

## 🧠 메모리 관리의 장점

| 특징 | 설명 |
|------|------|
| **안전성** | 런타임에 발생할 수 있는 메모리 오류(예: dangling pointer, buffer overflow)를 컴파일 타임에 방지 |
| **효율성** | GC 없이도 메모리 관리가 가능 |
| **성능** | 메모리 할당/해제가 컴파일 타임에 최적화됨 |

---

## ✅ 요약

Rust는 **컴파일 타임에 메모리 관리를 강제**함으로써, 다음과 같은 장점을 제공합니다:

- 메모리 누수 방지
- 데이터 경쟁 방지
- 런타임 오버헤드 없음
- 안전한 동시성 프로그래밍

이러한 메커니즘은 C/C++처럼 직접적인 메모리 제어를 제공하면서도, **안전성을 보장**하는 데 초점을 맞춥니다.
