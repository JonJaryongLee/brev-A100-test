# Prompt
Rust에서 메모리 관리는 어떻게 이루어지는지 설명해줘.

# Answer
Rust는 메모리 관리를 **컴파일 타임에 처리**하는 언어로, 프로그래머가 직접 메모리를 관리하지 않아도 안전하게 메모리를 사용할 수 있도록 설계되어 있습니다. 이는 C/C++과 같은 언어와 비교했을 때 **메모리 안전성**을 보장하면서도 **성능 저하 없이** 작동할 수 있게 해줍니다.

---

## 🔑 Rust의 메모리 관리 핵심 원칙

### 1. **소유권 시스템 (Ownership System)**

Rust의 핵심 개념 중 하나입니다. 각 값은 **한 번만 소유자(Owner)를 가질 수 있으며**, 소유자는 그 값을 관리합니다.

#### 주요 규칙:
- 각 값은 하나의 소유자만 가질 수 있다.
- 소유자가 범위를 벗어나면 값은 자동으로 해제된다 (Drop).
- 소유권은 이동(_MOVE)되며, 복사(COPY)는 특정 타입에 한해서만 발생한다.

#### 예시:

```rust
let s1 = String::from("hello");
let s2 = s1; // s1의 소유권이 s2로 이동됨
// println!("{}", s1); // 컴파일 오류! s1은 더 이상 유효하지 않음
```

---

### 2. **대여 시스템 (Borrowing)**

소유권을 변경하지 않고 값에 접근할 수 있는 방법입니다. `&T` (불변 대여) 또는 `&mut T` (가변 대여)를 통해 가능합니다.

#### 예시:

```rust
fn main() {
    let mut s = String::from("hello");
    change(&mut s); // 가변 대여
    println!("{}", s);
}

fn change(s: &mut String) {
    s.push_str(", world!");
}
```

#### 대여 규칙 (Borrowing Rules):
1. 하나의 가변 대여만 존재할 수 있음.
2. 여러 불변 대여는 동시에 존재 가능함.
3. 대여된 값은 소유자가 범위를 벗어나기 전까지는 사용 불가.

---

### 3. **생명주기 (Lifetime)**

Rust는 **생명주기**를 통해 참조자가 유효한 기간을 추적합니다. 이는 메모리 안정성을 보장하는 데 중요한 역할을 합니다.

#### 예시:

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

- `'a`는 두 입력 문자열 중 더 긴 문자열의 생명주기를 나타냄.

---

### 4. **Drop 트레이트**

Rust에서는 `Drop` 트레이트를 구현하여 **메모리 해제 시 실행될 코드**를 정의할 수 있습니다.

#### 예시:

```rust
struct MyStruct {
    data: String,
}

impl Drop for MyStruct {
    fn drop(&mut self) {
        println!("MyStruct is being dropped");
    }
}
```

---

## 🧠 메모리 관리의 장점

| 특징 | 설명 |
|------|------|
| **안전성** | 런타임 오류 없이 메모리 안전성을 보장 |
| **성능** | 컴파일 타임에 최적화되므로 런타임 오버헤드 없음 |
| **복잡성 감소** | GC가 필요 없고, 메모리 누수 방지 |

---

## 📌 요약

Rust는 다음과 같은 방식으로 메모리 관리를 수행합니다:

1. **소유권 시스템**으로 메모리의 소유자를 명확히 정의하고, 소유권 이동을 통해 메모리 관리.
2. **대여 시스템**으로 메모리에 대한 임시 접근을 허용하며, 런타임에 안전한 참조 관리.
3. **생명주기**로 참조자의 유효 기간을 컴파일 타임에 검증.
4. **Drop 트레이트**로 메모리 해제 시 추가 로직 실행 가능.

이러한 시스템 덕분에 Rust는 **C/C++처럼 빠르면서도 메모리 안전성을 보장**하는 언어로 자리 잡았습니다.
