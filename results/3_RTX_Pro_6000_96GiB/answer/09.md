# Prompt
PostgreSQL 데이터베이스 성능 최적화 방법을 알려줘.

# Answer
PostgreSQL 데이터베이스의 성능을 최적화하는 것은 애플리케이션의 응답 속도, 처리량, 자원 사용 효율성을 높이는 데 매우 중요합니다. 아래는 PostgreSQL 성능 최적화를 위한 주요 방법들입니다:

---

## 🔧 1. **인덱스(Index) 최적화**

### ✅ 인덱스 생성 전략:
- 자주 `WHERE`, `JOIN`, `ORDER BY` 절에서 사용되는 컬럼에 인덱스를 생성하세요.
- 복합 인덱스(Composite Index)는 여러 컬럼을 동시에 사용할 때 유용합니다.

```sql
CREATE INDEX idx_user_email ON users(email);
CREATE INDEX idx_composite ON orders(user_id, order_date);
```

### ⚠️ 주의사항:
- 너무 많은 인덱스는 `INSERT/UPDATE/DELETE` 성능을 저하시킬 수 있습니다.
- `SELECT`가 많고 `INSERT/UPDATE`가 적은 경우 인덱스를 많이 활용할 수 있습니다.

---

## 📊 2. **쿼리 최적화(Query Optimization)**

### ✅ 쿼리 실행 계획 확인:
- `EXPLAIN ANALYZE` 명령어로 쿼리 실행 계획을 분석하세요.

```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com';
```

### ✅ 조건절 최적화:
- `WHERE` 절에서 인덱스를 사용할 수 있도록 조건을 구성하세요.
- `LIKE` 연산자는 `%`로 시작하면 인덱스를 사용하지 못합니다.

```sql
-- 느림 (인덱스 미사용)
SELECT * FROM users WHERE name LIKE '%john%';

-- 빠름 (인덱스 사용 가능)
SELECT * FROM users WHERE name LIKE 'john%';
```

---

## 🛠️ 3. **설정 최적화 (postgresql.conf)**

### ✅ 주요 설정 항목:

| 설정 | 설명 |
|------|------|
| `shared_buffers` | PostgreSQL이 사용할 메모리 양 (권장: 전체 RAM의 25%) |
| `work_mem` | 각 작업(예: 정렬, 해시)에 할당되는 메모리 |
| `effective_cache_size` | OS 캐시 포함한 총 메모리 크기 |
| `maintenance_work_mem` | VACUUM, CREATE INDEX 등 작업 시 사용할 메모리 |
| `max_connections` | 동시 연결 수 제한 |

예시:
```conf
shared_buffers = 2GB
work_mem = 64MB
effective_cache_size = 6GB
maintenance_work_mem = 256MB
```

> 💡 설정 변경 후 PostgreSQL 재시작 필요

---

## 🧹 4. **VACUUM 및 ANALYZE 관리**

### ✅ VACUUM:
- 삭제된 행을 정리하고, 테이블의 공간을 회수합니다.

```sql
VACUUM users;
```

### ✅ ANALYZE:
- 테이블 통계 정보를 업데이트하여 쿼리 계획을 개선합니다.

```sql
ANALYZE users;
```

### ✅ 자동 VACUUM 설정:
PostgreSQL은 기본적으로 `autovacuum` 기능을 활성화합니다. 필요 시 설정 조정:

```conf
autovacuum = on
autovacuum_max_workers = 3
autovacuum_naptime = 10
```

---

## 🗃️ 5. **테이블 구조 최적화**

### ✅ 적절한 데이터 타입 사용:
- `VARCHAR(255)`보다는 `TEXT` 또는 `CHAR(10)` 같은 적절한 길이 사용.

### ✅ 정규화 vs 반정규화:
- JOIN이 많아지면 성능 저하 가능 → 필요 시 denormalization 적용.

### ✅ 파티셔닝 (Partitioning):
- 대용량 테이블은 날짜나 범주별로 파티셔닝하여 성능 향상.

```sql
CREATE TABLE orders_2024 PARTITION OF orders
FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');
```

## 📈 6. **모니터링 및 성능 분석 도구**

- **pg_stat_statements**: 자주 실행되는 쿼리 추적
- **pgBadger**: 로그 분석 도구
- **pgAdmin, psql**: 직접 쿼리 실행 및 성능 확인

```sql
-- 확장 설치
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;

-- 쿼리 통계 확인
SELECT query, calls, total_time, mean_time
FROM pg_stat_statements
ORDER BY total_time DESC;
```
