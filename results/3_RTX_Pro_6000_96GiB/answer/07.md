# Prompt
프로세스와 스레드의 차이점을 코드 레벨에서 설명해줘.

# Answer
프로세스(Process)와 스레드(Thread)는 운영체제에서 프로그램을 실행하는 두 가지 기본 단위입니다. 이 둘은 **실행 단위**로서의 역할은 비슷하지만, **메모리 공간과 자원 공유 방식**에서 큰 차이가 있습니다.

---

## 🔹 1. 정의

### 📌 프로세스 (Process)
- 실행 중인 프로그램의 인스턴스.
- 각각 **독립적인 메모리 공간**을 가진다.
- CPU 시간, 파일 디스크립터, 환경 변수 등 **자원을 독립적으로 관리**한다.

### 📌 스레드 (Thread)
- 프로세스 내부에서 실행되는 **최소 실행 단위**.
- 같은 프로세스 내의 **공유 메모리 공간**을 사용한다.
- 하나의 프로세스는 여러 스레드를 가질 수 있다.

---

## 🔹 2. 코드 레벨에서의 차이점

### ✅ 예시: C++에서 프로세스와 스레드 생성

#### 🧩 프로세스 생성 예시 (`fork()` 사용 - Unix/Linux 기준)

```cpp
#include <unistd.h>
#include <sys/wait.h>
#include <iostream>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 자식 프로세스
        std::cout << "Child process: PID = " << getpid() << std::endl;
    } else if (pid > 0) {
        // 부모 프로세스
        std::cout << "Parent process: PID = " << getpid() << std::endl;
        wait(NULL); // 자식 프로세스 종료 대기
    } else {
        std::cerr << "Fork failed!" << std::endl;
    }

    return 0;
}
```

- `fork()`는 새로운 프로세스를 생성하고, **메모리 공간을 복사**하여 독립된 프로세스를 만든다.
- 부모와 자식은 서로 다른 메모리 공간을 가지며, **변수 값도 서로 영향을 주지 않음**.

---

#### 🧩 스레드 생성 예시 (`pthread_create` 사용)

```cpp
#include <pthread.h>
#include <iostream>

void* thread_function(void* arg) {
    int id = *(int*)arg;
    std::cout << "Thread ID: " << id << std::endl;
    return nullptr;
}

int main() {
    pthread_t thread1, thread2;
    int id1 = 1, id2 = 2;

    pthread_create(&thread1, nullptr, thread_function, &id1);
    pthread_create(&thread2, nullptr, thread_function, &id2);

    pthread_join(thread1, nullptr);
    pthread_join(thread2, nullptr);

    return 0;
}
```

- `pthread_create()`는 **같은 프로세스 내에서 새로운 스레드를 생성**함.
- 스레드들은 **동일한 메모리 공간**을 공유하므로, 전역 변수나 동적 할당된 메모리는 **모두 접근 가능**하다.

---

## 🔹 3. 메모리 및 자원 공유

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 | 공유 |
| CPU 자원 | 독립적 | 공유 |
| 커널 리소스 | 각각 관리 | 공유 |
| 생성/종료 비용 | 높음 | 낮음 |
| 통신 방식 | IPC (예: 파이프, 메시지 큐) | 공유 메모리 |

---

## 🔹 4. 실제 예시: 메모리 공유

```cpp
#include <iostream>
#include <thread>

int shared_var = 0;

void increment() {
    for (int i = 0; i < 1000000; ++i) {
        shared_var++; // 공유 메모리에 접근
    }
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);

    t1.join();
    t2.join();

    std::cout << "Shared variable: " << shared_var << std::endl;
    return 0;
}
```

- `shared_var`는 **두 스레드가 공유**하므로, 경쟁 조건(Race Condition)이 발생할 수 있음.
- 프로세스에서는 `fork()` 이후에는 메모리가 복사되므로, **각각의 프로세스는 독립된 메모리 공간**을 가지므로 이런 문제가 발생하지 않음.
