# Prompt
Rust에서 메모리 관리는 어떻게 이루어지는지 설명해줘.

# Answer
Rust는 메모리 관리를 **컴파일 타임에 수행**하는 언어로, 프로그래머가 직접 메모리를 관리하지 않아도 안전하게 메모리를 사용할 수 있도록 설계되어 있습니다. 이는 C/C++과 같은 언어와 비교했을 때 **메모리 안전성과 성능**을 동시에 달성하기 위한 중요한 특징입니다.

---

## 🔧 Rust의 메모리 관리 방식

### 1. **소유권 시스템 (Ownership System)**

Rust의 핵심 개념 중 하나는 **소유권**(ownership)입니다. 이 시스템은 메모리 관리를 컴파일 타임에 처리하여 런타임 오버헤드 없이 메모리 안전성을 보장합니다.

#### 주요 규칙:
- 각 값은 **하나의 소유자**(owner)만 가질 수 있다.
- 소유자는 **하나의 값만 소유**할 수 있다.
- 소유자가 범위를 벗어나면(예: 함수가 끝나면), 해당 값은 자동으로 해제된다.

#### 예시:

```rust
let s1 = String::from("hello");
let s2 = s1; // s1의 소유권이 s2로 이동됨
// println!("{}", s1); // 컴파일 에러! s1은 더 이상 유효하지 않음
```

---

### 2. **대여자 시스템 (Borrowing System)**

소유권을 이전하지 않고 값을 참조할 수 있는 방법입니다. 이는 ** borrowing** 을 통해 구현됩니다.

#### 예시:

```rust
fn main() {
    let s = String::from("hello");
    let len = calculate_length(&s); // s를 참조함 (borrowing)
    println!("Length of {} is {}", s, len);
}

fn calculate_length(s: &String) -> usize {
    s.len() // s는 불변 참조자 (immutable reference)
}
```

- `&s`는 **불변 참조자**(immutable reference)입니다.
- `&mut s`는 **가변 참조자**(mutable reference)입니다.

#### 참조자의 규칙:
- 항상 하나 이상의 불변 참조자 또는 하나의 가변 참조자를 가질 수 있음.
- **가변 참조자는 하나만 존재 가능**하며, 그 동안 다른 모든 참조자는 존재할 수 없음.

---

### 3. **RAII (Resource Acquisition Is Initialization)**

Rust는 RAII 패턴을 사용하여 리소스(예: 메모리)를 자동으로 관리합니다. 이는 **Drop trait**을 통해 구현됩니다.

#### 예시:

```rust
struct MyStruct {
    data: Vec<i32>,
}

impl Drop for MyStruct {
    fn drop(&mut self) {
        println!("MyStruct is being dropped!");
    }
}
```

- `MyStruct` 인스턴스가 스코프를 벗어날 때 `drop` 메서드가 자동 호출됩니다.

---

### 4. **컴파일 타임 검사**

Rust 컴파일러는 메모리 관련 문제를 **컴파일 타임에 감지**합니다. 예를 들어:

- **중복된 소유권 이전** (double free)
- **사용 후 메모리 접근** (use after free)
- **경쟁 조건**(race condition)이 아닌 경우에도 메모리 안전성을 보장

## 🧠 요약

| 개념 | 설명 |
|------|------|
| **소유권** | 메모리의 소유자를 명확히 정의하여 메모리 누수 방지 |
| **대여자** | 소유권을 이전하지 않고 메모리 참조 |
| **RAII** | 리소스 관리에 대한 자동화 (Drop trait 활용) |
| **컴파일 타임 검사** | 런타임 오류 없이 메모리 안전성 보장 |

---

## ✅ 결론

Rust는 **컴파일 타임에 메모리 안전성을 보장**하는 언어입니다. 이는 **C/C++처럼 직접 메모리를 관리해야 하는 번거로움 없이**, **안전하고 빠른 코드 작성이 가능**하도록 도와줍니다. 이는 Rust가 시스템 프로그래밍 언어로 적합한 이유 중 하나입니다.

필요하다면 Rust의 메모리 모델에 대해 더 깊이 있게 설명해드릴 수 있어요!
