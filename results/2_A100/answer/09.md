# Prompt
PostgreSQL 데이터베이스 성능 최적화 방법을 알려줘.

# Answer
PostgreSQL 데이터베이스의 성능을 최적화하는 것은 애플리케이션의 응답 속도, 처리량, 자원 사용 효율성을 높이는 데 매우 중요합니다. 아래는 PostgreSQL 성능 최적화를 위한 주요 방법들을 정리한 것입니다:

---

## 🔧 1. **인덱스(Index) 최적화**

### ✅ 인덱스 생성 전략
- **WHERE 절에 자주 사용되는 컬럼**에 인덱스를 생성하세요.
- **SELECT, JOIN, ORDER BY, GROUP BY**에서 사용되는 컬럼에 인덱스를 적용하세요.

```sql
CREATE INDEX idx_user_email ON users(email);
```

### ✅ 복합 인덱스 (Composite Index)
- 여러 컬럼을 함께 사용할 경우, 순서가 중요한 복합 인덱스를 사용하세요.

```sql
CREATE INDEX idx_order_date_status ON orders(order_date, status);
```

### ✅ 인덱스 유형 선택
- B-tree: 기본 인덱스, 대부분의 쿼리에 적합
- Hash: 동일한 값 찾기(=)에 빠름 (단, PostgreSQL에서는 `hash` 인덱스는 일부 제한이 있음)
- GiST/GIN: 공간 데이터나 텍스트 검색에 유용

---

## 📊 2. **쿼리 최적화**

### ✅ EXPLAIN ANALYZE로 쿼리 분석
- 실행 계획을 확인하여 느린 쿼리를 파악하세요.

```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com';
```

### ✅ WHERE 조건 최적화
- 인덱스가 있는 컬럼을 기준으로 WHERE 절을 구성하세요.
- `LIKE` 연산은 `%value%` 형태는 인덱스를 사용하지 않으므로 주의!

### ✅ JOIN 최적화
- JOIN 시에는 **INNER JOIN**보다 **LEFT JOIN**이 더 빠를 수 있습니다.
- JOIN 순서와 조건을 잘 설계하세요.

---

## ⚙️ 3. **설정 최적화 (postgresql.conf)**

### ✅ 주요 설정 항목

| 설정 | 설명 |
|------|------|
| `shared_buffers` | PostgreSQL이 사용할 메모리 양 (권장: 전체 RAM의 25%) |
| `work_mem` | 각 작업에 할당되는 메모리 (쿼리당 사용) |
| `effective_cache_size` | OS 캐시 포함한 총 메모리 (권장: 전체 RAM의 50~75%) |
| `maintenance_work_mem` | VACUUM, ALTER TABLE 등 작업에 사용되는 메모리 |
| `checkpoint_completion_target` | 체크포인트 시간 조절 (0.8~0.9 권장) |

예시:
```conf
shared_buffers = 2GB
work_mem = 64MB
effective_cache_size = 6GB
```

> 💡 설정 변경 후 PostgreSQL 재시작 필요

## 🧹 4. **VACUUM 및 ANALYZE 관리**

### ✅ VACUUM
- 삭제된 행을 정리하고, 테이블의 공간을 회수합니다.

```sql
VACUUM users;
```

### ✅ ANALYZE
- 테이블 통계 정보를 갱신하여 쿼리 계획기를 최적화합니다.

```sql
ANALYZE users;
```

### ✅ 자동 VACUUM 설정
- `autovacuum` 옵션을 활성화하여 자동으로 정리되도록 합니다.

```conf
autovacuum = on
```

---

## 🗃️ 5. **테이블 구조 최적화**

### ✅ 적절한 데이터 타입 사용
- `VARCHAR(255)`보다는 `TEXT`, `CHAR(10)` 같은 적절한 크기 사용
- `INTEGER`보다 `SMALLINT` 또는 `BIGINT` 사용 (필요 시)

### ✅ 정규화 vs 반정규화
- JOIN이 많아지면 성능 저하 가능 → 필요 시 반정규화 고려

### ✅ Partitioning (분할 테이블)
- 대용량 테이블은 **PARTITIONING**을 통해 성능 향상 가능

```sql
CREATE TABLE orders_2023 PARTITION OF orders
FOR VALUES FROM ('2023-01-01') TO ('2024-01-01');
```

## 🔄 6. **캐싱 및 쿼리 캐싱**

- **pgBouncer**, **Redis**, **Application Level Caching** 등을 활용해 반복 쿼리 결과를 캐시하세요.
- 캐시는 DB 부하를 줄이고 응답 속도를 높입니다.

## 📈 7. **모니터링 도구 활용**

- **pg_stat_statements**: 가장 느린 쿼리 추적
- **pgAdmin**, **pgBadger**, **Prometheus + Grafana** 등을 활용하여 성능 모니터링

```sql
-- pg_stat_statements 활성화
LOAD 'pg_stat_statements';
```
