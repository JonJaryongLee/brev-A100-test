# Prompt
프로세스와 스레드의 차이점을 코드 레벨에서 설명해줘.

# Answer
프로세스(Process)와 스레드(Thread)는 운영체제에서 프로그램을 실행하는 두 가지 기본 단위입니다. 이 둘은 **실행 단위**로서의 역할은 비슷하지만, **메모리 공간과 자원 공유 방식**에서 큰 차이가 있습니다.

---

## 🔹 1. 정의

### 📌 프로세스 (Process)
- 실행 중인 프로그램의 인스턴스.
- 각각 **독립된 메모리 공간**을 가진다.
- CPU 시간, 파일 디스크립터, 환경 변수 등 **자원을 독립적으로 관리**한다.

### 📌 스레드 (Thread)
- 프로세스 내부에서 실행되는 **최소 실행 단위**.
- 같은 프로세스 내의 **공유 메모리 공간**을 사용한다.
- 하나의 프로세스는 여러 스레드를 가질 수 있다.

---

## 🔹 2. 코드 레벨에서의 차이 예시

### ✅ 예시: Python에서 프로세스와 스레드 비교

#### 1. **스레드 사용 (threading 모듈)**

```python
import threading
import time

def worker(name):
    for i in range(3):
        print(f"{name} working...")
        time.sleep(1)

# 스레드 생성
t1 = threading.Thread(target=worker, args=("Thread-1",))
t2 = threading.Thread(target=worker, args=("Thread-2",))

# 스레드 시작
t1.start()
t2.start()

# 스레드 종료 대기
t1.join()
t2.join()
```

- `t1`과 `t2`는 같은 프로세스 내에서 실행되며, **공유 메모리 공간**을 사용함.
- `print()` 같은 출력은 동일한 프로세스 내에서 공유됨.

#### 2. **프로세스 사용 (multiprocessing 모듈)**

```python
import multiprocessing
import time

def worker(name):
    for i in range(3):
        print(f"{name} working...")
        time.sleep(1)

# 프로세스 생성
p1 = multiprocessing.Process(target=worker, args=("Process-1",))
p2 = multiprocessing.Process(target=worker, args=("Process-2",))

# 프로세스 시작
p1.start()
p2.start()

# 프로세스 종료 대기
p1.join()
p2.join()
```

- `p1`과 `p2`는 **각각 다른 메모리 공간**을 가지며, **독립적인 프로세스**로 실행됨.
- 메모리와 자원은 서로 분리되어 있음.

---

## 🔹 3. 주요 차이 요약

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 | 공유 |
| 자원 공유 | 없음 | 공유 가능 |
| 생성/종료 비용 | 높음 | 낮음 |
| 컨텍스트 스위칭 | 비쌈 | 적음 |
| 병렬성 | 실제 병렬 처리 가능 (CPU 코어 수만큼) | 한 프로세스 내에서는 동시성만 가능 |
| 예시 언어 | C, Python, Java 등 | Python, Java 등 |

---

## 🔹 4. 실제 예시: 메모리 공유

```python
import threading

shared_var = 0

def increment():
    global shared_var
    for _ in range(100000):
        shared_var += 1

# 스레드에서 공유 변수 변경
t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)

t1.start()
t2.start()

t1.join()
t2.join()

print(shared_var)  # 예상치 못한 결과가 나올 수 있음 (race condition)
```

- `shared_var`는 **스레드 간에 공유**되므로, 경쟁 조건(Race Condition)이 발생할 수 있음.

---

## ✅ 결론

- **프로세스**: 각각의 메모리 공간을 가지며, 자원을 완전히 분리함 → 안정성 높음, 자원 소비 큼  
- **스레드**: 같은 메모리 공간을 공유하므로, 빠르고 효율적이지만, 동기화 문제 발생 가능성 있음  

코드 레벨에서 보면, **스레드는 같은 프로세스 내에서 실행되며**, **공유 메모리를 통해 데이터를 주고받는 반면**, **프로세스는 완전히 독립적인 메모리 영역**을 갖습니다.

필요하다면 C/C++ 또는 Java 코드로도 설명해 드릴 수 있어요!
