# Prompt
프로세스와 스레드의 차이점을 코드 레벨에서 설명해줘.

# Answer
프로세스(Process)와 스레드(Thread)는 운영체제에서 프로그램을 실행하는 두 가지 기본 단위입니다. 이 둘은 **실행 단위**로서의 역할은 비슷하지만, **메모리 공간과 자원 공유 방식**에서 큰 차이가 있습니다.

---

## 🔹 1. 정의

### 📌 프로세스 (Process)
- 실행 중인 프로그램의 인스턴스.
- 각각 **독립된 메모리 공간**을 가진다.
- CPU 시간, 파일 디스크립터, 환경 변수 등 **자신만의 자원**을 갖는다.

### 📌 스레드 (Thread)
- 프로세스 내부에서 실행되는 **최소 실행 단위**.
- 같은 프로세스 내의 **공유 메모리 공간**을 사용한다.
- 하나의 프로세스는 여러 스레드를 가질 수 있다.

---

## 🔹 2. 코드 레벨에서의 차이점

### ✅ 예시: C++에서 프로세스와 스레드 생성

#### 🧩 프로세스 생성 예시 (`fork()` 사용 - Unix/Linux 기준)

```cpp
#include <iostream>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 자식 프로세스
        std::cout << "Child process: PID = " << getpid() << std::endl;
        return 0;
    } else if (pid > 0) {
        // 부모 프로세스
        wait(NULL); // 자식 프로세스 종료 대기
        std::cout << "Parent process: PID = " << getpid() << std::endl;
    } else {
        std::cerr << "Fork failed!" << std::endl;
    }

    return 0;
}
```

- `fork()`는 새로운 프로세스를 생성하고, **완전히 독립된 메모리 공간**을 가진다.
- 부모와 자식은 서로 다른 메모리 영역을 사용하므로, 한쪽에서 변수를 수정해도 다른 쪽에는 영향 없음.

---

#### 🧩 스레드 생성 예시 (`pthread` 사용)

```cpp
#include <iostream>
#include <pthread.h>

void* thread_function(void* arg) {
    int id = *(int*)arg;
    std::cout << "Thread ID: " << id << std::endl;
    return nullptr;
}

int main() {
    pthread_t thread1, thread2;

    int id1 = 1, id2 = 2;

    pthread_create(&thread1, nullptr, thread_function, &id1);
    pthread_create(&thread2, nullptr, thread_function, &id2);

    pthread_join(thread1, nullptr);
    pthread_join(thread2, nullptr);

    return 0;
}
```

- `pthread_create()`는 **같은 프로세스 내에서 스레드를 생성**함.
- 스레드들은 **공유 메모리 공간**을 사용하므로, 전역 변수나 동일한 메모리 영역에 접근 가능.

---

## 🔹 3. 주요 차이 요약

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 | 공유 |
| 자원 공유 | 불가능 | 가능 |
| 생성/종료 비용 | 높음 | 낮음 |
| 컨텍스트 스위칭 비용 | 높음 | 낮음 |
| 통신 방식 | IPC (Inter-Process Communication) | 공유 메모리 |
| 예시 | `fork()`, `exec()` | `pthread_create()` |

---

## 🔹 4. 실제 예시: 메모리 공유

```cpp
#include <iostream>
#include <pthread.h>

int shared_var = 0;

void* increment_thread(void* arg) {
    for (int i = 0; i < 5; ++i) {
        shared_var++;
        std::cout << "Thread: " << shared_var << std::endl;
    }
    return nullptr;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, nullptr, increment_thread, nullptr);
    pthread_create(&t2, nullptr, increment_thread, nullptr);
    pthread_join(t1, nullptr);
    pthread_join(t2, nullptr);
    return 0;
}
```

- 두 스레드는 `shared_var`를 **공유**하여 접근하고 수정함 → **竞合 조건**(race condition) 발생 가능성 있음.

---

## ✅ 결론

- **프로세스**는 **완전히 독립적인 실행 단위**로, 메모리 공간이 분리되어 있어 보안성과 안정성이 높지만, 자원 소비가 많고 커뮤니케이션 비용이 큼.
- **스레드**는 **공유 메모리를 통해 실행되는 단위**로, 빠르고 효율적이지만, 동기화 문제(예: 데드락, 경쟁 상태)가 발생할 수 있음.
