# Prompt
Rust에서 메모리 관리는 어떻게 이루어지는지 설명해줘.

# Answer
Rust는 메모리 관리를 **컴파일 타임에 처리**하는 언어로, 프로그래머가 직접 메모리를 관리하지 않아도 안전하게 메모리를 사용할 수 있도록 설계되어 있습니다. 이는 C/C++과 같은 언어와 비교했을 때 **메모리 안전성**과 **성능**을 동시에 달성하기 위한 중요한 특징입니다.

---

## 🔑 Rust의 메모리 관리 원리

Rust의 메모리 관리는 다음과 같은 핵심 개념들에 의해 이루어집니다:

### 1. **소유권 (Ownership)**

- Rust의 핵심 개념으로, 각 값은 하나의 **소유자**(owner)만 가질 수 있습니다.
- 소유자는 해당 값을 **소유하고 있는 변수**입니다.
- 소유권이 이동되면 원래 변수는 더 이상 사용할 수 없습니다.

```rust
let s1 = String::from("hello");
let s2 = s1; // s1의 소유권이 s2로 이동됨
// println!("{}", s1); // 컴파일 오류! s1은 더 이상 유효하지 않음
```

### 2. **대여 (Borrowing)**

- 소유권을 넘기지 않고 **참조**를 통해 값을 사용할 수 있습니다.
- `&T` (불변 참조) 또는 `&mut T` (가변 참조)로 표현됩니다.

```rust
fn main() {
    let s = String::from("hello");
    let len = calculate_length(&s); // s를 불변 참조로 전달
    println!("Length: {}", len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

### 3. **Lifetime**

- 참조자의 생존 기간을 컴파일 타임에 검사하여 **dangling pointer** 문제를 방지합니다.
- `&'a T`와 같이 lifetime 파라미터를 사용하여 참조의 유효기간을 명시합니다.

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

---

## 🧠 메모리 관리의 장점

1. **메모리 누수 방지**: 소유권 시스템 덕분에 메모리 누수를 컴파일 타임에 방지할 수 있습니다.
2. **데이터 경쟁 방지**: 런타임에 발생할 수 있는 데이터 경쟁을 컴파일 타임에 감지할 수 있습니다.
3. **성능**: Rust는 C/C++과 비슷한 성능을 제공하면서도 메모리 안전성을 보장합니다.

---

## 📌 요약

| 개념 | 설명 |
|------|------|
| **소유권** | 값의 소유자가 누구인지 정의하고, 이동/복사 규칙을 적용 |
| **대여** | 소유권 없이 값에 접근 가능 (참조) |
| **Lifetime** | 참조의 유효 기간을 컴파일 타임에 검사 |
| **Compile-time safety** | 메모리 안전성을 컴파일 타임에 보장 |

---

Rust는 이러한 메커니즘들을 통해 **안전성과 효율성**을 동시에 달성하며, 메모리 관리를 프로그래머가 직접 하지 않아도 되도록 도와줍니다.
