# GPU 성능 테스트 결과 분석 보고서

- 테스트 일자: 2026.03.19
- 테스트 담당자: 웹개발2팀 이자룡

본 보고서는 NVIDIA의 세 가지 GPU 모델을 대상으로 실시한 LLM 추론 성능 테스트 결과입니다.  

## 1. 테스트 개요
- **사용한 모델**
  - 4비트 양자화한 Qwen3-coder 
- **대상 하드웨어**:
  1. NVIDIA L40S (48GiB)
  2. NVIDIA RTX Pro 6000 (96GiB)
  3. NVIDIA H100 (80GiB)
- **테스트 시나리오**:
  - 총 여섯개 테스트 시나리오입니다.
    - 단일 요청 (디코딩/인코딩)
    - 10개 동시 요청 (디코딩/인코딩)
    - 30개 동시 요청 (디코딩/인코딩)

## 2. 하드웨어별 상세 분석

### 2.1 NVIDIA L40S (48GiB)
L40S 모델은 테스트된 세 모델 중 상대적으로 가장 낮은 성능을 보였습니다. 
- **단일 처리**: 초당 약 180 토큰 내외의 디코딩 성능을 기록했습니다.
- **동시 처리**: 요청 수가 10개에서 30개로 늘어남에 따라 전체 처리량(TPS)은 약 820에서 2000 수준으로 증가했으나, 개별 요청당 처리 속도는 감소하는 경향을 보였습니다.
- **특이사항**: 가성비 측면에서는 유리할 수 있으나, 서비스 제공하는 입장에선 하한선이라고 봅니다.

### 2.2 NVIDIA RTX Pro 6000 (96GiB)
가장 큰 비디오 메모리(96GiB)를 탑재한 모델로, 특히 고부하(30개 동시 요청) 상황에서 뛰어난 성능을 보였습니다.
- **단일 처리**: 디코딩 약 191 TPS, 인코딩 약 187 TPS로 안정적인 성능을 기록했습니다.
- **동시 처리**: 30개 동시 요청 시 전체 처리량이 약 2,700~2,800 TPS에 달해, 본 테스트 환경에서는 H100보다도 높은 총 처리량을 기록하기도 했습니다.
- **특이사항**: 넉넉한 VRAM 덕분에 대규모 동시 요청 처리 시 병목 현상이 적게 발생하는 것으로 분석됩니다.

### 2.3 NVIDIA H100 (80GiB)
개별 요청 및 중급 부하(10개 동시 요청)에서 가장 우수한 효율을 보였습니다.
- **단일 처리**: RTX Pro 6000과 큰 차이 없는 성능을 보였습니다.
- **동시 처리**: 10개 동시 요청 시 평균 119 TPS로 가장 높은 효율을 기록했습니다. 30개 동시 요청 시에도 안정적이었으나, 총 처리량 측면에서는 RTX Pro 6000 의 VRAM 이 더 많아서그런지 약간 모자랍니다.
- **특이사항**: 셋 중 가장 강력한 연산 성능을 보입니다. VRAM 이 모자란게 살짝 아쉽습니다.

## 3. 종합 비교 분석

| 지표 (평균 TPS) | L40S (48GiB) | RTX Pro 6000 (96GiB) | H100 (80GiB) |
| :--- | :---: | :---: | :---: |
| 단일 디코딩 | ~181 | **~191** | 191 |
| 10개 동시 (평균) | ~82 | ~114 | **~119** |
| 30개 동시 (평균) | ~67 | **~92** | ~82 |

### 3.1 확장성 평가
- RTX Pro 6000 은 VRAM 이 커서그런지 30개 동시 요청 시 가장 높은 확장성을 보여주었습니다.
- H100 은 10개 동시 요청 상황에서 최고의 효율을 보이며, 연산 속도가 가장 빠릅니다.
- 10번의 요청 처리를 가정하면 H100, 30번의 요청 처리를 가정하면 RTX Pro 6000 이 더 낫습니다.

### 3.2 VRAM 관련
테스트 결과, 단순 아키텍처 성능뿐만 아니라 VRAM 용량이 동시 처리 성능에 중요함을 확인할 수 있었습니다. 96GiB를 탑재한 RTX Pro 6000이 고부하 상황에서 H100(80GiB)을 앞서는 결과를 보입니다.

## 4. 결론
1. 30명 이상 많은 유저가 동시 접속한다면 VRAM 이 강점인 RTX Pro 6000 96GiB
2. 10명이 동시 요청을 한다 가정했을 때 개별 최고 성능은 H100
3. 돈 좀 아낀다면 L40S
